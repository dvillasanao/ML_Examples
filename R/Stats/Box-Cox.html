Transformaci√≥n Box-Cox

Apuntes y anotaciones personales

Author

Diana Villasana Ocampo

La Transformaci√≥n Box-Cox es una t√©cnica estad√≠stica utilizada para transformar datos no normales para que se aproximen a una distribuci√≥n normal. Esto es crucial en muchas t√©cnicas estad√≠sticas que asumen que los datos (o los errores del modelo) se distribuyen normalmente, como la regresi√≥n lineal, el an√°lisis de varianza (ANOVA) y algunas gr√°ficas de control.

Principales usos

1. Normalizaci√≥n de los datos:

  • Muchos modelos estad√≠sticos, especialmente los param√©tricos, asumen que los datos siguen una distribuci√≥n normal. Si los datos est√°n sesgados (es decir, no son sim√©tricos, con una ‚Äúcola‚Äù m√°s larga hacia un lado) o presentan una distribuci√≥n no normal, aplicar la transformaci√≥n Box-Cox puede ayudar a que se parezcan m√°s a una campana de Gauss.

  • Esto permite que las pruebas de hip√≥tesis y los intervalos de confianza derivados de estos modelos sean m√°s v√°lidos y precisos.

2. Estabilizaci√≥n de la varianza (Homocedasticidad):

  • En muchos an√°lisis, como la regresi√≥n lineal, se asume que la varianza de los errores es constante en todos los niveles de las variables predictoras (homocedasticidad).

  • Si la varianza cambia a medida que los valores de la variable aumentan o disminuyen (heterocedasticidad), los resultados del modelo pueden ser ineficientes o sesgados.

  • La transformaci√≥n Box-Cox puede ayudar a estabilizar esta varianza, haciendo que el modelo sea m√°s robusto y sus estimaciones m√°s confiables.

3. Mejora de la linealidad:

  • En la regresi√≥n lineal, se asume una relaci√≥n lineal entre la variable dependiente y las variables independientes. Si la relaci√≥n no es lineal, transformar la variable dependiente (o incluso las independientes, con la transformaci√≥n Box-Tidwell) puede ayudar a establecer una relaci√≥n m√°s lineal, mejorando el ajuste del modelo.

4. Facilita la interpretaci√≥n y el an√°lisis:

  • Al normalizar y estabilizar los datos, los patrones subyacentes se vuelven m√°s discernibles y f√°ciles de analizar.
  • Tambi√©n puede reducir la influencia de valores at√≠picos y extremos, lo que puede distorsionar los resultados.

Encontrar \(\lambda\)

La transformaci√≥n Box-Cox no es un c√°lculo directo y √∫nico, sino un proceso para encontrar el valor √≥ptimo del par√°metro \(\lambda\) que mejor normaliza los datos. Una vez que se encuentra ese \(\lambda\) √≥ptimo, se aplica la f√≥rmula de la transformaci√≥n a los datos.

El m√©todo m√°s com√∫n para encontrar el \(\lambda\) √≥ptimo es el de M√°xima Verosimilitud (Maximum Likelihood Estimation - MLE). Aqu√≠ te explico los pasos generales:

  1. Define la funci√≥n de verosimilitud: Para cada posible valor de \(\lambda\), se transforma la variable \(Y\) (los datos) utilizando la f√≥rmula de Box-Cox:
  • Si \(\lambda \neq 0\): \(Y^{(\lambda)} = \frac{Y^\lambda - 1}{\lambda}\)
  • Si \(\lambda = 0\): \(Y^{(\lambda)} = \log(Y)\)

Luego, se asume que la variable transformada \(Y^{(\lambda)}\) sigue una distribuci√≥n normal. La funci√≥n de verosimilitud para una distribuci√≥n normal es:

\[L(\lambda, \mu, \sigma^2 | Y) = \prod_{i=1}^{n} \frac{1}{\sqrt{2\pi\sigma^2}} \exp\left(-\frac{(Y_i^{(\lambda)} - \mu)^2}{2\sigma^2}\right) \times J(\lambda, Y)\]

Donde:

  • \(Y_i^{(\lambda)}\) es la \(i\)-√©sima observaci√≥n transformada.
  • \(\mu\) y \(\sigma^2\) son la media y la varianza de la variable transformada (que tambi√©n deben estimarse para cada \(\lambda\)).
  • \(J(\lambda, Y)\) es el jacobiano de la transformaci√≥n, que es esencial para asegurar que la probabilidad total se mantenga constante despu√©s de la transformaci√≥n. Para la transformaci√≥n Box-Cox, el jacobiano es \(\prod_{i=1}^{n} Y_i^{\lambda-1}\).

En la pr√°ctica, se suele trabajar con el logaritmo de la funci√≥n de verosimilitud (log-verosimilitud) porque es m√°s f√°cil de maximizar.

  1. Iterar sobre un rango de valores de \(\lambda\): Se prueba un rango de valores para \(\lambda\) (por ejemplo, de -2 a 2, o -1 a 1, con incrementos peque√±os como 0.1 o 0.01).

  2. Para cada \(\lambda\):

  • Transforma los datos \(Y\) usando el \(\lambda\) actual.
  • Calcula la media (\(\mu\)) y la varianza (\(\sigma^2\)) de los datos transformados.
  • Calcula el valor de la funci√≥n de log-verosimilitud para ese \(\lambda\).

  1. Selecciona el \(\lambda\) √≥ptimo: El valor de \(\lambda\) que produce la m√°xima log-verosimilitud es considerado el \(\lambda\) √≥ptimo para transformar los datos.

  2. Aplica la transformaci√≥n: Una vez que tienes el \(\lambda\) √≥ptimo, aplicas la transformaci√≥n Box-Cox con ese \(\lambda\) a todos los datos originales.

library(ggplot2)
library(dplyr) # Para manipulaci√≥n de datos
library(tidyr) # Para pivot_longer
library(latex2exp)

# 1. Crear un conjunto de datos original de ejemplo
# Usaremos un vector de valores positivos para demostrar la transformaci√≥n.
# Aqu√≠, usaremos un rango simple para visualizar el efecto.
original_data <- data.frame(x = seq(0.1, 10, length.out = 100) # Evitar 0 para Box-Cox
)

# 2. Definir una funci√≥n Box-Cox simple
# Esta funci√≥n aplica la transformaci√≥n Box-Cox para un dado lambda
box_cox_transform <- function(y, lambda) {
  if (lambda == 0) {
    log(y)
  } else {
    (y^lambda - 1) / lambda
  }
}

# 3. Generar las transformaciones para diferentes lambdas
# Creamos un vector de lambdas para visualizar
lambdas_to_plot <- c(-2, -1, -0.5, 0, 0.5, 1, 2)

# Aplicar cada transformaci√≥n y almacenar los resultados
transformed_data <- original_data %>%
  rowwise() %>% # Aplicar la funci√≥n por fila
  mutate(
    across(.cols = x,
           .fns = setNames(
             lapply(lambdas_to_plot, function(l) ~ box_cox_transform(., l)),
             paste0("lambda_", lambdas_to_plot)
           ),
           .names = "{.fn}")
  ) %>%
  ungroup()

# 4. Transformar los datos a formato "largo" para ggplot2
# Esto es necesario para graficar m√∫ltiples l√≠neas por lambda
plot_data <- transformed_data %>%
              pivot_longer(
                           cols = starts_with("lambda_"),
                           names_to = "lambda_value",
                           names_prefix = "lambda_",
                           values_to = "transformed_x"
                          ) %>%
               mutate(
                      # Convertir el valor de lambda a factor para que ggplot2 lo trate como discreto
                      # y para un mejor orden de las leyendas.
                      lambda_value = factor(lambda_value, levels = as.character(lambdas_to_plot))
                     )

# 5. Generar el gr√°fico ggplot2
ggplot(plot_data, aes(x = x, y = transformed_x, color = lambda_value)) +
 geom_line(size = 1) +
  labs(
       title = "Efecto de la Transformaci√≥n Box-Cox para Diferentes Valores de Lambda",
       x = "Valor Original (x)",
       y = TeX("Valor Transformado ($x^\\lambda$)"),
       color = expression(lambda) # Usar expresi√≥n para el s√≠mbolo lambda en la leyenda
       ) +
  theme_minimal() +
   theme(
        plot.title = element_text(hjust = 0.5, face = "bold"),
        legend.position = "right"
        ) +
    scale_color_viridis_d(option = "plasma", end = 0.9) + 
     facet_wrap(.~lambda_value, scales = "free")
